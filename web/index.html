Why mocli

Focused, not bloated

Only the Microsoft workflows needed for daily automation: communications, schedules, tasks, and files.

Agent-first execution

JSON-first output, stable exit codes, and non-interactive flags keep orchestration reliable.

Capabilities with realistic examples

Mail

mo mail list --max 10
mo mail send \
  --to ops-team@outlook.com \
  --subject "Action items from Mar 7 incident review" \
  --body "Please close remaining P1 follow-ups by EOD Friday."

Calendar

mo calendar create \
  --summary "Infra Postmortem: API Rate Limit Event" \
  --from 2026-03-09T17:00:00Z \
  --to 2026-03-09T17:45:00Z \
  --description "Owners, timeline, and corrective actions."

Tasks

mo tasks create \
  --title "Prepare weekly revenue digest for leadership" \
  --body "Include NA/EU breakdown and variance notes."
mo tasks update <task-id> --status inProgress --importance high

OneDrive

mo drive upload ./q1-finance-summary.pdf --conflict rename
mo drive share <item-id> --to user --email cfo.office@outlook.com --role read
mo drive download <item-id> --out ./archive/q1-finance-summary.pdf
